# Paramètres du modèle d’alerte personnalisé

| Onglet | Paramètre | Description |
|--- |--- |--- |
| [!UICONTROL Date Range] | [!UICONTROL Date] | Période pour laquelle évaluer les conditions de l’alerte. Les mesures sont évaluées sous forme d’agrégats sur toute la période. Les options sont comprises entre *[!UICONTROL Yesterday]* et *[!UICONTROL Last 180 Days]*. |
| [!UICONTROL Filters] |  | Conditions de déclenchement de l’alerte au moment spécifié dans l’onglet [!UICONTROL Scheduling and Delivery]. Les colonnes disponibles varient en fonction du type d’entité (campagne ou mot-clé, par exemple). Tous les filtres sont unis à l&#39;aide de la fonction booléenne AND, ce qui signifie que toutes les conditions spécifiées doivent être remplies.<ul><li><p>Pour ajouter un filtre, cliquez sur ![Flèche vers le bas](/help/search-social-commerce/assets/arrow-down-expand.png "Flèche vers le bas") en regard de ![Ajouter](/help/search-social-commerce/assets/add.png "Ajouter") **[!UICONTROL ADD FILTER]**, puis procédez comme suit :</p></li><ol><li><p>(Facultatif) Pour filtrer les noms de colonne par chaîne de texte, saisissez la chaîne de recherche dans le champ d’entrée **[!UICONTROL ADD FILTER]**.</p></li><li><p>Sélectionnez un nom de colonne dans le menu Colonnes.</p></li><li><p>Définissez le filtre sur la colonne :</p></li><ul><li><p>(Filtres sans champs de saisie) Cliquez sur ![Flèche vers le bas](/help/search-social-commerce/assets/arrow-down-expand.png "Flèche vers le bas") en regard du second menu, puis cochez les cases en regard de chaque valeur à inclure.</p></li><li><p>(Colonnes de mesures pour lesquelles vous souhaitez suivre les augmentations ou les diminutions de valeur entre la période spécifiée et la période précédente) Pour l’opérateur, sélectionnez *augmente de* ou *diminue de*, puis saisissez la valeur de seuil.</p><p>Par exemple, si vous avez sélectionné la colonne &quot;[!UICONTROL Cost]&quot; et souhaitez être alerté lorsque le coût d’une campagne augmente de 21 %, sélectionnez &quot;[!UICONTROL increases by]&quot; et saisissez 21 dans le champ de saisie. Dans le champ [!UICONTROL Date Comparison Format] , indiquez que vous êtes intéressé par un changement de pourcentage (plutôt qu’un changement des unités monétaires).</p><p>Lorsque vous sélectionnez cette option, deux colonnes supplémentaires sont ajoutées pour chaque colonne de données ordinaire. Par exemple, au lieu d’inclure une seule colonne pour &quot;Clics&quot;, le rapport inclut des colonnes pour &quot;Clics R1&quot; (pour la plage 1) &quot;Clics R2&quot; (pour la plage 2), et &quot;Clics Diff&quot; ou &quot;ClicsDiff (%)&quot;, selon le [!UICONTROL Date Comparison Format] spécifié (voir ci-dessous).</p><p>**Notes :**</p><ul><li><p>La colonne de différence n’est pas renseignée pour les mesures dérivées.<p></li><li><p>Cette fonctionnalité n’est pas disponible pour les colonnes de classification de mesure, d’ID ou d’étiquette de conversion.</p></li><li><p>La génération de rapports qui comparent des plages de dates volumineuses peut prendre plus de temps.</p></li></ul></ul><li><p>(Tous les autres filtres avec champs de saisie) Sélectionnez un opérateur dans le deuxième menu, puis saisissez la valeur appropriée.</p><p>Par exemple, si vous avez sélectionné la colonne &quot;Clics&quot; et souhaitez ne renvoyer que les lignes comportant plus de 100 clics, sélectionnez &quot;supérieur à&quot; et saisissez 100 dans le champ de saisie.</p><p>Selon le type de données, les opérateurs disponibles peuvent inclure <i>supérieur à</i>, <i>inférieur à</i>, <i>égal à</i>, <i>contient</i>, <i>ne contient pas</i>, <i>commence par</i>, <i>se termine par</i>, <i>aucune valeur</i>&rbrace;, <i> a la valeur </i>, <i>before</i>, <i>after</i> ou <i>no date</i>.</p><p>**Remarque :** Les valeurs de texte ne sont pas sensibles à la casse. Par exemple, si vous filtrez par campagnes dont le nom contient &quot;prêt&quot;, les résultats peuvent inclure &quot;prêts aux consommateurs&quot; et &quot;demandes de prêt&quot;.</p></li></ol><li><p>Pour supprimer un filtre, cliquez sur ![Supprimer](/help/search-social-commerce/assets/delete.png "Supprimer") en regard de la définition de filtre.</p></li></ul> |
|  | [!UICONTROL Comparing] | (Disponible lorsque l’alerte suit une augmentation ou une diminution d’une ou de plusieurs colonnes de mesures ; lecture seule) Les deux plages de dates pour lesquelles les données sont comparées. |
|  | [!UICONTROL Date Comparison Format] | (Disponible lorsque l’alerte suit une augmentation ou une diminution d’une ou de plusieurs colonnes de mesures) Comment exprimer la différence entre les données des deux périodes :<ul><li><p><i>[!UICONTROL Variance]</i> (valeur par défaut) : affiche la différence sous la forme d’une valeur numérique.</p></li><li><p><i>[!UICONTROL % Change]</i> — Affiche la différence en pourcentage.</p></li></ul> |
| [!UICONTROL Scheduling and Delivery] | [!UICONTROL Name] | Nom de l’alerte. Elle doit contenir au moins cinq caractères. |
|  | [!UICONTROL Trigger this Alert] [when] | Fréquence à laquelle l’alerte recherche les filtres de condition spécifiés et, lorsque toutes les conditions sont remplies, envoie des notifications par courrier électronique :<ul><li><p>[!UICONTROL Daily at <*NN*> [AM|PM]]</p></li><li><p>[!UICONTROL Weekly on <*Day of Week*> at <*NN*> [AM|PM]]</p></li><li><p>[!UICONTROL Every month on <*Day NN*> at <*NN*> [AM|PM]]</p></li></ul>**Remarque :** Cette valeur n’affecte pas la période d’évaluation. |
|  | [!UICONTROL Email Recipients] | (Modifiable uniquement par le créateur du modèle d’alerte ; en lecture seule pour tous les autres) Adresses électroniques à partir desquelles envoyer des notifications lorsqu’une alerte est générée. Par défaut, l&#39;adresse du créateur du modèle est renseignée.<br><br>Pour ajouter une adresse, saisissez l’adresse, puis cliquez sur **[!UICONTROL Add]**. Pour spécifier plusieurs adresses, séparez-les par des virgules ou des espaces, ou ajoutez-les séparément.<br><br> Lorsque l’alerte comprend jusqu’à 1 000 enregistrements, une version CSV de l’alerte est jointe au message électronique. |

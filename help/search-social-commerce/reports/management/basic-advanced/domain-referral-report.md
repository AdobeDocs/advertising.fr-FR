# [!UICONTROL Domain Referral Report]

<!-- If we remove this report, also remove concept topic "Domain Optimization." -->

La [!UICONTROL Domain Referral Report] inclut l’estimation du nombre de clics payants [^1] par nom de domaine pour les campagnes dans des portfolios ou des comptes publicitaires spécifiés (le cas échéant). Vous pouvez éventuellement inclure des mesures de conversion. Par défaut, les données incluent le nombre estimé de clics pour chaque combinaison groupe publicitaire/campagne/portfolio pour chaque domaine qui a généré des clics, et les lignes sont dans l’ordre décroissant des clics estimés.

Les impressions et le coût estimés ne sont disponibles que pour les campagnes de suivi d’affichage uniquement.

Vous pouvez afficher les données des deux (2) derniers mois plus le mois en cours.

>[!NOTE]
>
>Les données sont disponibles pour les campagnes de recherche et d’affichage uniquement.

>[!TIP]
>
>Utilisez les résultats pour affiner les paramètres réseau de vos campagnes. Par exemple, si les publicités sur les sites partenaires d’un réseau publicitaire ou sur un site web spécifique ne produisent jamais de conversions, vous pouvez exclure les publicités de leur exécution. Toutefois, faites attention lorsque vous excluez des sites de vos campagnes, car les changements de contenu et de marché peuvent rapidement modifier la valeur du trafic d’un site.

[^1] : Les données de clics (ou clics publicitaires) de ces rapports sont présentées sous forme d’estimations plutôt que de données réelles parce que Search, Social et Commerce peuvent utiliser des algorithmes différents de ceux des différents réseaux publicitaires pour détecter des clics valides, et parce que Search, Social et Commerce ne comptabilisent pas les mots-clés pour lesquels aucun clic n’a été enregistré la semaine précédente. Le système de suivi des conversions peut également inclure des données pour les comptes de réseau publicitaire avec lesquels Search, Social et Commerce ne se synchronisent pas.
